# 基于多Agent交叉验证的可信威胁情报分析系统 —— 演示文档

---

## 一、系统概览

本系统通过**三阶段多Agent流水线**实现威胁情报的可信分析：检索Agent（FAISS向量库 + Neo4j知识图谱）→ 联网验证Agent（Tavily搜索）→ 综合分析Agent（结构化对比输出）。系统同时提供纯LLM和单Agent ReAct两种对照模式。

## 三、三模式交叉纠正演示

### 案例1 — T0034：2021年7月全球勒索攻击利用哪家厂商漏洞？

| 模式 | 准确率 | 幻觉 | 知识锚 | 回答摘要 |
|------|:---:|:---:|:---:|------|
| 纯LLM | 1 | **是** | 否 | 声称是"Citrix ADC CVE-2019-19781"，编造REvil利用Citrix的虚假事件 |
| 单Agent | 1 | **是** | 否 | 指向"微软Exchange ProxyShell漏洞"，混淆WannaCry/SolarWinds事件 |
| **多Agent** | **5** | **否** | **是** | 联网搜索独立确认→**Kaseya VSA平台漏洞**，攻击者为REvil，赎金7000万美元 [来源: W2, W3] |

**纠正机制**：本地库无相关记录 → 联网搜索独立获取Kaseya/REvil供应链攻击信息 → 综合Agent标注为"Web补充"

## 六、低准确率根因分析（多Agent模式 acc≤2 的 68 题）

## 八、技术栈

| 组件 | 技术选型 |
|------|------|
| 大语言模型 | Qwen3.6-27B-FP8 (vLLM推理) |
| 向量库 | FAISS + BGE-large-zh-v1.5 (1024维) |
| 混合检索 | BM25 + FAISS → RRF融合 → BGE-Reranker精排 |
| 知识图谱 | Neo4j (200+节点，1-3跳查询) |
| Agent框架 | LangGraph StateGraph (手写，不依赖create_agent) |
| 联网搜索 | Tavily Search API (并行3 query) |
| 流式输出 | asyncio.Queue + ContextVar (绕过checkpoint序列化) |
| 评测框架 | LLM-as-Judge 五维评分 + 3模式并行评测 |
